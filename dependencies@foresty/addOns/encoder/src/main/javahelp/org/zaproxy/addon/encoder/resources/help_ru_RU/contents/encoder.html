Диалоговое окно кодирования/декодирования/хеширования

Это позволяет кодировать, декодировать или хешировать текст.

Общие поля

В диалоговом окне есть одно поле, общее для всех вкладок:

Текст для кодирования/декодирования/хеширования:

Это поле предназначено для текста, который вы хотите закодировать, декодировать или хэшировать.
Если какой-либо текст выделен при вызове диалога, он будет помещен в это поле. Другие поля будут обновляться динамически, если вы измените текст

Панель инструментов

  • Добавить новую вкладку — добавляет новую вкладку в диалоговое окно.
  • Удалить выбранную вкладку — удаляет выбранную (или текущую) вкладку из диалогового окна.
  • Удалить выбранную вкладку — удаляет выбранную (или текущую) вкладку из диалогового окна.
  • Сброс — сброс всех вкладок/панелей до состояния по умолчанию.

Предопределенные процессоры кодирования/декодирования/хеширования

Кодировщики

2 байтовый недопустимый UTF8

Будет отображаться с недопустимой последовательностью символов UTF-8 с 2 байтами.

3-байтовый недопустимый UTF8

Будет отображаться недопустимая последовательность символов UTF-8 с 3 байтами.

4 байтовый недопустимый UTF8

Отобразит недопустимую последовательность символов UTF-8 с 4 байтами.

Шестнадцатиричное кодирование ASCII

Будет отображаться шестнадцатеричная кодировка ASCII вводимого текста.

Кодировка Base 64

Будет отображаться кодировка base 64 введенного вами текста.

URL-кодирование Base 64

Будет отображаться базовая кодировка URL-адреса 64 для введенного вами текста. Base64URL — это модификация основного стандарта base 64. целью которого является возможность использовать результат кодирования в качестве имени файла или URL-адреса. Base64URL описан в RFC 4648, в котором отмечается, что стандартный алфавит с основанием 64 включает символы, недопустимые для URL-адреса и имена файлов (например, знак плюс и вопросительный знак являются проблематичными). Основной стандарт будет кодировать <<???>> в PDw/Pz8+Pg==, а Base64URL преобразует его в PDw_Pz8-Pg. .

Экранированный текст Unicode

Отображает экранированные символы Unicode. Например, текст Açores будет закодирован как %u0041%u00e7%u006f%u0072%u0065%u0073.

HTML кодирование

Будет отображаться кодировка HTML введенного вами текста. Например, текст "piñata" будет закодирован как &quot;pi&ntilde;ata&quot;.

Кодирование JavaScript

Отображает экранированные литералы JavaScript для введенного вами текста. Например, текст "2￠" будет закодирован как \"2\uFFE0\".

Кодирование URL

Отобразит URL-кодировку вводимого текста.

Полное кодирование URL

Будет отображаться полная кодировка URL-адреса введенного вами текста (все символы преобразованы в HEX и добавлены проценты).

Неэкранированный текст Unicode

Отображает неэкранированные символы Unicode. Например, текст %u0041%u00e7%u006f%u0072%u0065%u0073 будет расшифрован как Açores.

Декодеры

Шестнадцатиричное декодирование ASCII

Будет отображаться шестнадцатеричное кодирование ASCII введенного текста.
Если действительного декодирования нет, то поле будет отключено.

Декодирование Base 64

Отобразит расшифровку вводимого текста в формате base 64.

Декодирование URL-адресов Base 64

Отобразит декодирование URL-адреса base 64 введенного вами текста. Base64URL — это модификация основного стандарта base 64. целью которого является возможность использовать результат кодирования в качестве имени файла или URL-адреса. Base64URL описан в RFC 4648, в котором отмечается, что стандартный алфавит с основанием 64 включает символы, недопустимые для URL-адреса и имена файлов (например, знак плюс и вопросительный знак являются проблематичными).

Декодирование HTML

Отобразит HTML-декодирование введенного вами текста. Например, текст &quot;pi&ntilde;ata&quot; будет расшифрован как "piñata".

Декодирование JavaScript

Отображает неэкранированные литералы JavaScript вводимого вами текста. Например, текст \"2\uFFE0\" будет расшифрован как "2￠".

Декодирование URL

Отобразит расшифровку URL-адреса введенного вами текста.

Полное декодирование URL

Будет отображаться декодирование URL-адреса введенного вами текста (знаки процента удалены и декодировано HEX).

Хэшеры

Хэширование MD5

Отобразит хеш MD5 введенного текста.

Хэш SHA1

Отобразит хэш SHA1 введенного вами текста.

Хэш SHA256

Отобразит хэш SHA256 введенного вами текста.

Custom (Индивидуальное Создание)

Пользовательские сценарии «Кодировать/Декодировать» можно создавать/добавлять в дерево сценариев и использовать в настраиваемых панелях вывода.
