Поддержка инфраструктуры автоматизации GraphQL

Это дополнение поддерживает Automation Framework.

Надстройка будет импортировать схемы GraphQL с помощью самоанализа, если во время поиска будут найдены конечные точки. Однако рекомендуется указать конечную точку и (необязательно) файл схемы или URL-адрес, если они доступны.

Задание: graphql

Задание graphql позволяет импортировать схемы GraphQL локально или по URL-адресу. Он поддерживает следующие параметры: 
  - type: graphql                      # Импорт определения GraphQL 
    parameters:
      endpoint:                        # String: URL-адрес конечной точки, по умолчанию: null, схема не импортируется. 
      schemaUrl:                       # String: URL-адрес, указывающий на схему GraphQL, по умолчанию: null, импорт с использованием самоанализа на конечной точке 
      schemaFile:                      # String: Путь к локальному файлу схемы GraphQL, по умолчанию: null, импорт с использованием schemaUrl 
      maxQueryDepth:                   # Int: Максимальная глубина генерации запроса, по умолчанию: 5 
      lenientMaxQueryDepthEnabled:     # Bool: Независимо от того, применяется ли максимальная глубина запроса снисходительно, по умолчанию: true 
      maxAdditionalQueryDepth:         # Int: Максимальная дополнительная глубина генерации запроса (используется, если применяется снисходительно), по умолчанию: 5 
      maxArgsDepth:                    # Int: Максимальная глубина генерации аргументов, по умолчанию: 5 
      optionalArgsEnabled:             # Bool: Должны ли быть указаны необязательные аргументы, по умолчанию: true 
      argsType:                        # Enum [inline, variables, both]: Как указываются аргументы, по умолчанию: оба 
      querySplitType:                  # Enum [leaf, root_field, operation]: Уровень, для которого генерируется один запрос, по умолчанию: leaf 
      requestMethod:                   # Enum [post_json, post_graphql, get]: Метод запроса по умолчанию: post_json
