Поддержка GraphQL

Это дополнение позволяет импортировать определения GraphQL и отправлять сгенерированные на их основе запросы.

Надстройка автоматически обнаружит любые определения GraphQL и просканирует их, пока они находятся в области действия.
Паук поддерживается в ZAP 2.10.0 и более поздних версиях.

Интерфейс

В меню Импорт добавлены 2 пункта меню:
  • Импорт схемы GraphQL из файла
  • Импорт схемы GraphQL из URL-адреса

Формат URL-адреса конечной точки

URL конечной точки имеет следующий формат:
scheme://authority/path
со всеми компонентами URI, обязательными при импорте из файла.

Панель параметров

Панель параметров GraphQL добавлена ​​в меню «Инструменты» -> «Параметры» -> «GraphQL». Эти параметры позволяют управлять выводом Генератора запросов.

API

В API добавлены следующие операции:

Просмотры

  • optionArgsType
  • optionMaxArgsDepth
  • optionMaxQueryDepth
  • optionOptionalArgsEnabled
  • optionQuerySplitType
  • optionRequestMethod

Действия

  • importFile (endurl*, file*)
  • importUrl (endurl*, url)
  • setOptionArgsType (String*)
  • setOptionMaxArgsDepth (Integer*)
  • setOptionMaxQueryDepth (Integer*)
  • setOptionOptionalArgsEnabled (Boolean*)
  • setOptionQuerySplitType (String*)
  • setOptionRequestMethod (String*)
endurl supports the Endpoint URL формат, описанный ранее. Определения будут импортированы синхронно, и будут возвращены все предупреждения.

командная строка

Добавлены следующие параметры командной строки:
  • -graphqlfile <filename>: импортирует определение GraphQL из файла.
  • -graphqlurl <url>: импортирует определение GraphQL из URL-адреса.
  • -graphqlendurl <url> : устанавливает URL-адрес конечной точки.
Определения будут импортированы синхронно, а все предупреждения будут отображаться в командной строке.
