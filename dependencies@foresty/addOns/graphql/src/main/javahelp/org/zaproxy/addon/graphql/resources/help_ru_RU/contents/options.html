Параметры GraphQL

В этом документе «запрос» может относиться к запросу GraphQL, подписке или мутации.

Максимальная глубина запроса

Максимальная глубина генерируемого запроса.

Мягкая максимальная глубина запроса

Если этот параметр включен, он предотвращает неверные запросы, разрешая дополнительную глубину для полей без конечных типов. Если этот параметр отключен, максимальная глубина запроса применяется строго, даже если это означает создание недопустимого запроса.

Дополнительная глубина запроса

Значение этого параметра используется, только если включен параметр «Максимальная глубина запроса». Максимальная дополнительная глубина, используемая для поиска полей листового типа. Если конечный тип не найден, сгенерированный запрос может быть недопустимым, и в журнал записывается сообщение.

Максимальная глубина аргументов

Максимальная глубина указанных аргументов. Это полезно, когда поле имеет входной объект в качестве аргумента.

Укажите необязательные аргументы

Если выбран этот параметр, то также будут указаны необязательные (обнуляемые) аргументы.

Укажите аргументы

Выберите, как аргументы поля должны быть добавлены в сгенерированный запрос,
  • В линию
  • Использование переменных
  • Оба (Both) — каждый запрос будет отправлен дважды, один раз со встроенными аргументами и один раз с переменными.

Создать запрос для

Отдельный запрос может быть сгенерирован для любого из следующего в предоставленной схеме:
  • каждый лист, то есть каждый скаляр или перечисление;
  • каждое поле под корневым типом операции (запрос, мутация или подписка); или
  • каждый тип корневой операции.

Метод запроса

Запросы к конечной точке могут быть следующих типов:
  • Запрос POST с телом JSON (тип содержимого: application/json)
  • Запрос POST с запросом GraphQL в теле (Content-Type: application/graphql)
  • Запрос GET с добавленным к нему запросом в строке запроса
