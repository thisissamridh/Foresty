Поддержка платформы автоматизации фильтра предупреждений

Это дополнение поддерживает платформу автоматизации.

Работа: alertFilter

Задание alertFilter позволяет определять глобальные и контекстно-зависимые фильтры предупреждений. 
  - type: alertFilter # Используется для изменения уровней риска предупреждений.
    параметры:
      deleteGlobalAlerts: true # Boolean, если true, то будут удалены все существующие глобальные оповещения, по умолчанию false
    alertFilters: # Список применяемых фильтров alertFilters
      - ruleId: # Int: идентификатор правила обязательного оповещения.
        newRisk: # String: Обязательный новый уровень риска, один из «Ложно-положительный», «Информация», «Низкий», «Средний», «Высокий».
        context: # String: Необязательное имя контекста, если оно пусто, будет создан глобальный фильтр предупреждений
        url: # String: Необязательная строка для сопоставления с предупреждением, поддерживает переменные среды
        urlRegex: # Boolean: Необязательно, если true, то URL-адрес является регулярным выражением
        параметр: # String: Необязательная строка для сопоставления с полем параметра предупреждения
        parameterRegex: # Boolean: Необязательно, если true, то параметр является регулярным выражением, поддерживает переменные среды
        Атака: # Строка: Необязательная строка для сопоставления с полем атаки предупреждения
        attackRegex: # Boolean: Необязательно, если true, то атака является регулярным выражением
        Доказательство: # Строка: Необязательная строка для сопоставления с полем свидетельства предупреждения
        ДоказательствоRegex: # Boolean: Необязательно, если true, то доказательством является регулярное выражение.
