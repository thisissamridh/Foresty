Автоматизация Платформа - Работа ActiveScan

Это задание управляет активным сканером. Это активно атакует ваши приложения и поэтому следует использовать только против приложений, которые у вас есть разрешение на тестирование.

По умолчанию это задание будет активно сканировать первый контекст, определенный в среде, поэтому ни один из параметров не является обязательным.

YAML

  - type: activeScan                   # Активный сканер — он активно атакует цель, поэтому его следует использовать только с разрешения. 
    parameters:
      context:                         # Строка: имя контекста для атаки, по умолчанию: первый контекст 
      user:                            # Строка: необязательный пользователь для аутентификации должен быть определен в env. 
      policy:                          # Строка: имя используемой политики сканирования, по умолчанию: политика по умолчанию. 
      maxRuleDurationInMins:           # Int: максимальное время в минутах, в течение которого может выполняться любое отдельное правило, по умолчанию: 0 неограниченно 
      maxScanDurationInMins:           # Int: максимальное время в минутах, в течение которого активный сканер может работать, по умолчанию: 0 неограниченно 
      addQueryParam:                   # Логический (Bool): если установлено, будет добавлен дополнительный параметр запроса к запросам, у которых его нет, по умолчанию: false 
      defaultPolicy:                   # Строка (String): имя используемой политики сканирования по умолчанию, по умолчанию: политика по умолчанию. 
      delayInMs:                       # Int: задержка в миллисекундах между каждым запросом, используемая для снижения нагрузки на цель, по умолчанию 0 
      handleAntiCSRFTokens:            # Bool: Если установлено, то автоматически обрабатывать токены защиты от CSRF, по умолчанию: false 
      injectPluginIdInHeader:          # Bool: Если установлено, соответствующий идентификатор правила будет вставлен в заголовок X-ZAP-Scan-ID каждого запроса, по умолчанию: false

Данные задания

Следующий класс будет доступен для надстроек, предоставляющих доступ к данным задания, таких как надстройка Reporting. Обратите внимание, что в этом случае данные относятся к последнему активному сканированию, независимо от того, было ли оно запущено с помощью Automation Framework, пользовательского интерфейса или API.
