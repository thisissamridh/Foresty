Платформа автоматизации — работа запрашивающей стороны

Это задание отправляет специально созданные запросы на целевой URL-адрес с настраиваемым методом и телом запроса. Пользователь также может указать код ожидаемого ответа, с которым сравнивается фактический ответ, и пользователь получает предупреждение, если он не совпадает.

YAML

  - type: requestor                    # Используется для отправки определенных запросов к целям 
    parameters:
      user:                            # String: Необязательный пользователь, который будет использоваться для аутентифицированных запросов, должен быть определен в env. 
    requests:                          # Список запросов, которые нужно сделать 
      - url:                           # String: Обязательный URL запроса, который необходимо сделать 
        method:                        # String: Непустой метод запроса, по умолчанию: GET 
        data:                          # String: Необязательные данные для отправки в теле запроса 
        responseCode:                  # Int: Необязательный ожидаемый код ответа, с которым будет сравниваться фактический код ответа.
